24  /**
25   * Factory that creates instances of DateTimeFormatter based on the ISO8601 standard.
26   * <p>
27   * Date-time formatting is performed by the {@link DateTimeFormatter} class.
28   * Three classes provide factory methods to create formatters, and this is one.
29   * The others are {@link DateTimeFormat} and {@link DateTimeFormatterBuilder}.
30   * <p>
31   * ISO8601 is the international standard for data interchange. It defines a
32   * framework, rather than an absolute standard. As a result this provider has a
33   * number of methods that represent common uses of the framework. The most common
34   * formats are {@link #date() date}, {@link #time() time}, and {@link #dateTime() dateTime}.
35   * <p>
36   * For example, to format a date time in ISO format:
37   * <pre>
38   * DateTime dt = new DateTime();
39   * DateTimeFormatter fmt = ISODateTimeFormat.dateTime();
40   * String str = fmt.print(dt);
41   * </pre>
42   * <p>
43   * Note that these formatters mostly follow the ISO8601 standard for printing.
44   * For parsing, the formatters are more lenient and allow formats that are not
45   * in strict compliance with the standard.
46   * <p>
47   * It is important to understand that these formatters are not linked to
48   * the <code>ISOChronology</code>. These formatters may be used with any
49   * chronology, however there may be certain side effects with more unusual
50   * chronologies. For example, the ISO formatters rely on dayOfWeek being
51   * single digit, dayOfMonth being two digit and dayOfYear being three digit.
52   * A chronology with a ten day week would thus cause issues. However, in
53   * general, it is safe to use these formatters with other chronologies.
54   * <p>
55   * ISODateTimeFormat is thread-safe and immutable, and the formatters it
56   * returns are as well.
57   *
58   * @author Brian S O'Neill
59   * @since 1.0
60   * @see DateTimeFormat
61   * @see DateTimeFormatterBuilder
62   */
63  public class ISODateTimeFormat {

75      /**
76       * Returns a formatter that outputs only those fields specified.
77       * <p>
78       * This method examines the fields provided and returns an ISO-style
79       * formatter that best fits. This can be useful for outputting
80       * less-common ISO styles, such as YearMonth (YYYY-MM) or MonthDay (--MM-DD).
81       * <p>
82       * The list provided may have overlapping fields, such as dayOfWeek and
83       * dayOfMonth. In this case, the style is chosen based on the following
84       * list, thus in the example, the calendar style is chosen as dayOfMonth
85       * is higher in priority than dayOfWeek:
86       * <ul>
87       * <li>monthOfYear - calendar date style
88       * <li>dayOfYear - ordinal date style
89       * <li>weekOfWeekYear - week date style
90       * <li>dayOfMonth - calendar date style
91       * <li>dayOfWeek - week date style
92       * <li>year
93       * <li>weekyear
94       * </ul>
95       * The supported formats are:
96       * <pre>
97       * Extended      Basic       Fields
98       * 2005-03-25    20050325    year/monthOfYear/dayOfMonth
99       * 2005-03       2005-03     year/monthOfYear
100      * 2005--25      2005--25    year/dayOfMonth *
101      * 2005          2005        year
102      * --03-25       --0325      monthOfYear/dayOfMonth
103      * --03          --03        monthOfYear
104      * ---03         ---03       dayOfMonth
105      * 2005-084      2005084     year/dayOfYear
106      * -084          -084        dayOfYear
107      * 2005-W12-5    2005W125    weekyear/weekOfWeekyear/dayOfWeek
108      * 2005-W-5      2005W-5     weekyear/dayOfWeek *
109      * 2005-W12      2005W12     weekyear/weekOfWeekyear
110      * -W12-5        -W125       weekOfWeekyear/dayOfWeek
111      * -W12          -W12        weekOfWeekyear
112      * -W-5          -W-5        dayOfWeek
113      * 10:20:30.040  102030.040  hour/minute/second/milli
114      * 10:20:30      102030      hour/minute/second
115      * 10:20         1020        hour/minute
116      * 10            10          hour
117      * -20:30.040    -2030.040   minute/second/milli
118      * -20:30        -2030       minute/second
119      * -20           -20         minute
120      * --30.040      --30.040    second/milli
121      * --30          --30        second
122      * ---.040       ---.040     milli *
123      * 10-30.040     10-30.040   hour/second/milli *
124      * 10:20-.040    1020-.040   hour/minute/milli *
125      * 10-30         10-30       hour/second *
126      * 10--.040      10--.040    hour/milli *
127      * -20-.040      -20-.040    minute/milli *
128      *   plus datetime formats like {date}T{time}
129      * </pre>
130      * * indiates that this is not an official ISO format and can be excluded
131      * by passing in <code>strictISO</code> as <code>true</code>.
132      * <p>
133      * This method can side effect the input collection of fields.
134      * If the input collection is modifiable, then each field that was added to
135      * the formatter will be removed from the collection, including any duplicates.
136      * If the input collection is unmodifiable then no side effect occurs.
137      * <p>
138      * This side effect processing is useful if you need to know whether all
139      * the fields were converted into the formatter or not. To achieve this,
140      * pass in a modifiable list, and check that it is empty on exit.
141      *
142      * @param fields  the fields to get a formatter for, not null,
143      *  updated by the method call unless unmodifiable,
144      *  removing those fields built in the formatter
145      * @param extended  true to use the extended format (with separators)
146      * @param strictISO  true to stick exactly to ISO8601, false to include additional formats
147      * @return a suitable formatter
148      * @throws IllegalArgumentException if there is no format for the fields
149      * @since 1.1
150      */
151     public static DateTimeFormatter forFields(
152         Collection<DateTimeFieldType> fields,
153         boolean extended,
154         boolean strictISO) {
155         
156         if (fields == null || fields.size() == 0) {
157             throw new IllegalArgumentException("The fields must not be null or empty");
158         }
159         Set<DateTimeFieldType> workingFields = new HashSet<DateTimeFieldType>(fields);
160         int inputSize = workingFields.size();
161         boolean reducedPrec = false;
162         DateTimeFormatterBuilder bld = new DateTimeFormatterBuilder();
163         // date
164         if (workingFields.contains(DateTimeFieldType.monthOfYear())) {
165             reducedPrec = dateByMonth(bld, workingFields, extended, strictISO);
166         } else if (workingFields.contains(DateTimeFieldType.dayOfYear())) {
167             reducedPrec = dateByOrdinal(bld, workingFields, extended, strictISO);
168         } else if (workingFields.contains(DateTimeFieldType.weekOfWeekyear())) {
169             reducedPrec = dateByWeek(bld, workingFields, extended, strictISO);
170         } else if (workingFields.contains(DateTimeFieldType.dayOfMonth())) {
171             reducedPrec = dateByMonth(bld, workingFields, extended, strictISO);
172         } else if (workingFields.contains(DateTimeFieldType.dayOfWeek())) {
173             reducedPrec = dateByWeek(bld, workingFields, extended, strictISO);
174         } else if (workingFields.remove(DateTimeFieldType.year())) {
175             bld.append(Constants.ye);
176             reducedPrec = true;
177         } else if (workingFields.remove(DateTimeFieldType.weekyear())) {
178             bld.append(Constants.we);
179             reducedPrec = true;
180         }
181         boolean datePresent = (workingFields.size() < inputSize);
182         
183         // time
184         time(bld, workingFields, extended, strictISO, reducedPrec, datePresent);
185         
186         // result
187         if (bld.canBuildFormatter() == false) {
188             throw new IllegalArgumentException("No valid format for fields: " + fields);
189         }
190         
191         // side effect the input collection to indicate the processed fields
192         // handling unmodifiable collections with no side effect
193         try {
194             fields.retainAll(workingFields);
195         } catch (UnsupportedOperationException ex) {
196             // ignore, so we can handle unmodifiable collections
197         }
198         return bld.toFormatter();
199     }

202     /**
203      * Creates a date using the calendar date format.
204      * Specification reference: 5.2.1.
205      *
206      * @param bld  the builder
207      * @param fields  the fields
208      * @param extended  true to use extended format
209      * @param strictISO  true to only allow ISO formats
210      * @return true if reduced precision
211      * @since 1.1
212      */
213     private static boolean dateByMonth(
214         DateTimeFormatterBuilder bld,
215         Collection<DateTimeFieldType> fields,
216         boolean extended,
217         boolean strictISO) {
218         
219         boolean reducedPrec = false;
220         if (fields.remove(DateTimeFieldType.year())) {
221             bld.append(Constants.ye);
222             if (fields.remove(DateTimeFieldType.monthOfYear())) {
223                 if (fields.remove(DateTimeFieldType.dayOfMonth())) {
224                     // YYYY-MM-DD/YYYYMMDD
225                     appendSeparator(bld, extended);
226                     bld.appendMonthOfYear(2);
227                     appendSeparator(bld, extended);
228                     bld.appendDayOfMonth(2);
229                 } else {
230                     // YYYY-MM/YYYY-MM
231                     bld.appendLiteral('-');
232                     bld.appendMonthOfYear(2);
233                     reducedPrec = true;
234                 }
235             } else {
236                 if (fields.remove(DateTimeFieldType.dayOfMonth())) {
237                     // YYYY--DD/YYYY--DD (non-iso)
238                     checkNotStrictISO(fields, strictISO);
239                     bld.appendLiteral('-');
240                     bld.appendLiteral('-');
241                     bld.appendDayOfMonth(2);
242                 } else {
243                     // YYYY/YYYY
244                     reducedPrec = true;
245                 }
246             }
247             
248         } else if (fields.remove(DateTimeFieldType.monthOfYear())) {
249             bld.appendLiteral('-');
250             bld.appendLiteral('-');
251             bld.appendMonthOfYear(2);
252             if (fields.remove(DateTimeFieldType.dayOfMonth())) {
253                 // --MM-DD/--MMDD
254                 appendSeparator(bld, extended);
255                 bld.appendDayOfMonth(2);
256             } else {
257                 // --MM/--MM
258                 reducedPrec = true;
259             }
260         } else if (fields.remove(DateTimeFieldType.dayOfMonth())) {
261             // ---DD/---DD
262             bld.appendLiteral('-');
263             bld.appendLiteral('-');
264             bld.appendLiteral('-');
265             bld.appendDayOfMonth(2);
266         }
267         return reducedPrec;
268     }

272      * Creates a date using the ordinal date format.
273      * Specification reference: 5.2.2.
274      *
275      * @param bld  the builder
276      * @param fields  the fields
277      * @param extended  true to use extended format
278      * @param strictISO  true to only allow ISO formats
279      * @since 1.1
280      */
281     private static boolean dateByOrdinal(
282         DateTimeFormatterBuilder bld,
283         Collection<DateTimeFieldType> fields,
284         boolean extended,
285         boolean strictISO) {
286         
287         boolean reducedPrec = false;
288         if (fields.remove(DateTimeFieldType.year())) {
289             bld.append(Constants.ye);
290             if (fields.remove(DateTimeFieldType.dayOfYear())) {
291                 // YYYY-DDD/YYYYDDD
292                 appendSeparator(bld, extended);
293                 bld.appendDayOfYear(3);
294             } else {
295                 // YYYY/YYYY
296                 reducedPrec = true;
297             }
298             
299         } else if (fields.remove(DateTimeFieldType.dayOfYear())) {
300             // -DDD/-DDD
301             bld.appendLiteral('-');
302             bld.appendDayOfYear(3);
303         }
304         return reducedPrec;
305     }

308     /**
309      * Creates a date using the calendar date format.
310      * Specification reference: 5.2.3.
311      *
312      * @param bld  the builder
313      * @param fields  the fields
314      * @param extended  true to use extended format
315      * @param strictISO  true to only allow ISO formats
316      * @since 1.1
317      */
318     private static boolean dateByWeek(
319         DateTimeFormatterBuilder bld,
320         Collection<DateTimeFieldType> fields,
321         boolean extended,
322         boolean strictISO) {
323         
324         boolean reducedPrec = false;
325         if (fields.remove(DateTimeFieldType.weekyear())) {
326             bld.append(Constants.we);
327             if (fields.remove(DateTimeFieldType.weekOfWeekyear())) {
328                 appendSeparator(bld, extended);
329                 bld.appendLiteral('W');
330                 bld.appendWeekOfWeekyear(2);
331                 if (fields.remove(DateTimeFieldType.dayOfWeek())) {
332                     // YYYY-WWW-D/YYYYWWWD
333                     appendSeparator(bld, extended);
334                     bld.appendDayOfWeek(1);
335                 } else {
336                     // YYYY-WWW/YYYY-WWW
337                     reducedPrec = true;
338                 }
339             } else {
340                 if (fields.remove(DateTimeFieldType.dayOfWeek())) {
341                     // YYYY-W-D/YYYYW-D (non-iso)
342                     checkNotStrictISO(fields, strictISO);
343                     appendSeparator(bld, extended);
344                     bld.appendLiteral('W');
345                     bld.appendLiteral('-');
346                     bld.appendDayOfWeek(1);
347                 } else {
348                     // YYYY/YYYY
349                     reducedPrec = true;
350                 }
351             }
352             
353         } else if (fields.remove(DateTimeFieldType.weekOfWeekyear())) {
354             bld.appendLiteral('-');
355             bld.appendLiteral('W');
356             bld.appendWeekOfWeekyear(2);
357             if (fields.remove(DateTimeFieldType.dayOfWeek())) {
358                 // -WWW-D/-WWWD
359                 appendSeparator(bld, extended);
360                 bld.appendDayOfWeek(1);
361             } else {
362                 // -WWW/-WWW
363                 reducedPrec = true;
364             }
365         } else if (fields.remove(DateTimeFieldType.dayOfWeek())) {
366             // -W-D/-W-D
367             bld.appendLiteral('-');
368             bld.appendLiteral('W');
369             bld.appendLiteral('-');
370             bld.appendDayOfWeek(1);
371         }
372         return reducedPrec;
373     }

376     /**
377      * Adds the time fields to the builder.
378      * Specification reference: 5.3.1.
379      * 
380      * @param bld  the builder
381      * @param fields  the fields
382      * @param extended  whether to use the extended format
383      * @param strictISO  whether to be strict
384      * @param reducedPrec  whether the date was reduced precision
385      * @param datePresent  whether there was a date
386      * @since 1.1
387      */
388     private static void time(
389         DateTimeFormatterBuilder bld,
390         Collection<DateTimeFieldType> fields,
391         boolean extended,
392         boolean strictISO,
393         boolean reducedPrec,
394         boolean datePresent) {
395         
396         boolean hour = fields.remove(DateTimeFieldType.hourOfDay());
397         boolean minute = fields.remove(DateTimeFieldType.minuteOfHour());
398         boolean second = fields.remove(DateTimeFieldType.secondOfMinute());
399         boolean milli = fields.remove(DateTimeFieldType.millisOfSecond());
400         if (!hour && !minute && !second && !milli) {
401             return;
402         }
403         if (hour || minute || second || milli) {
404             if (strictISO && reducedPrec) {
405                 throw new IllegalArgumentException("No valid ISO8601 format for fields because Date was reduced precision: " + fields);
406             }
407             if (datePresent) {
408                 bld.appendLiteral('T');
409             }
410         }
411         if (hour && minute && second || (hour && !second && !milli)) {
412             // OK - HMSm/HMS/HM/H - valid in combination with date
413         } else {
414             if (strictISO && datePresent) {
415                 throw new IllegalArgumentException("No valid ISO8601 format for fields because Time was truncated: " + fields);
416             }
417             if (!hour && (minute && second || (minute && !milli) || second)) {
418                 // OK - MSm/MS/M/Sm/S - valid ISO formats
419             } else {
420                 if (strictISO) {
421                     throw new IllegalArgumentException("No valid ISO8601 format for fields: " + fields);
422                 }
423             }
424         }
425         if (hour) {
426             bld.appendHourOfDay(2);
427         } else if (minute || second || milli) {
428             bld.appendLiteral('-');
429         }
430         if (extended && hour && minute) {
431             bld.appendLiteral(':');
432         }
433         if (minute) {
434             bld.appendMinuteOfHour(2);
435         } else if (second || milli) {
436             bld.appendLiteral('-');
437         }
438         if (extended && minute && second) {
439             bld.appendLiteral(':');
440         }
441         if (second) {
442             bld.appendSecondOfMinute(2);
443         } else if (milli) {
444             bld.appendLiteral('-');
445         }
446         if (milli) {
447             bld.appendLiteral('.');
448             bld.appendMillisOfSecond(3);
449         }
450     }
